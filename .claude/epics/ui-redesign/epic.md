# Epic: UI 重新设计

## Overview

全面重新设计 batana 棒球挥棒分析系统的用户界面，采用 Flutter-Neumorphic 设计风格，解决当前视觉过时、控件重叠、交互不友好的问题。本次重新设计将覆盖所有页面和模块，保留现有核心功能的同时，新增从相册选择视频进行分析的能力，为业余棒球爱好者提供科技艺术感的专业友好体验。

技术实施将采用组件化架构，建立统一的设计系统（Design System），确保界面一致性和可维护性。重点优化性能表现，确保界面响应时间 < 100ms，页面切换动画流畅（60fps）。

## Architecture Decisions

### AD1: 设计系统架构
**决策**：建立独立的设计系统模块（`lib/design_system/`）
**理由**：
- 统一管理颜色、字体、间距、组件样式
- 便于全局主题切换和维护
- 提高组件复用性，减少重复代码
- 支持未来深色模式扩展

**实施方式**：
```
lib/design_system/
├── colors.dart          # 色彩系统
├── typography.dart      # 字体系统
├── spacing.dart         # 间距系统
├── neumorphic_theme.dart # Neumorphic 主题配置
└── widgets/             # 通用组件库
    ├── buttons.dart
    ├── cards.dart
    └── progress_indicators.dart
```

### AD2: 状态管理方案
**决策**：继续使用 Provider 进行状态管理
**理由**：
- 项目已有 Provider 基础，无需引入新依赖
- 适合中小型应用的状态管理需求
- 学习曲线平缓，易于维护
- 性能满足当前需求

**关键状态**：
- `VideoSourceState`: 管理视频来源（录制/相册）
- `AnalysisProgressState`: 管理分析进度
- `HistoryState`: 管理历史记录

### AD3: 相册集成方案
**决策**：使用 `image_picker` 插件
**理由**：
- Flutter 官方推荐插件，维护活跃
- 同时支持 iOS 和 Android
- API 简洁，易于集成
- 支持视频选择和预览

**配置要求**：
- iOS: 添加 `NSPhotoLibraryUsageDescription` 权限
- Android: 添加 `READ_EXTERNAL_STORAGE` 权限

### AD4: 页面路由架构
**决策**：使用命名路由（Named Routes）
**理由**：
- 清晰的页面导航结构
- 便于深度链接和页面跳转
- 易于测试和维护

**路由定义**：
```dart
routes: {
  '/': HomeScreen,
  '/record': RecordScreen,
  '/analysis': AnalysisProgressScreen,
  '/result': ResultScreen,
  '/history': HistoryScreen,
  '/settings': SettingsScreen,
}
```

### AD5: 动画性能优化
**决策**：使用 `AnimatedBuilder` 和 `RepaintBoundary`
**理由**：
- 减少不必要的重绘，提升性能
- 确保 60fps 流畅动画
- 降低 CPU 和 GPU 负载

## Technical Approach

### 设计系统实施

#### 1. Neumorphic 主题配置
- 集成 `flutter_neumorphic` 包
- 定义主色调：科技蓝 + 温暖灰
- 配置阴影参数：深度、模糊、偏移
- 建立组件样式规范

#### 2. 组件库开发
- 基础组件：按钮、卡片、输入框
- 业务组件：视频预览卡、评分卡、进度指示器
- 布局组件：页面容器、导航栏、底部栏

### 页面重构策略

#### 1. 主界面（HomeScreen）
**当前问题**：布局混乱，功能入口不清晰
**重构方案**：
- 采用卡片式布局，清晰分区
- 三大功能入口：录制视频、选择相册、历史记录
- 快速访问最近分析结果
- 底部导航栏：主页、历史、设置

#### 2. 录制界面（RecordScreen）
**当前问题**：控件重叠，操作不直观
**重构方案**：
- 全屏相机预览
- 浮动控制按钮：开始/停止/重录
- 顶部状态栏：录制时长、网格开关
- 底部提示：录制指引

#### 3. 相册选择界面（GalleryPickerScreen）- 新增
**功能**：从相册选择视频进行分析
**实施方案**：
- 使用 `image_picker` 插件
- 视频列表展示：缩略图 + 时长 + 分辨率
- 支持视频预览
- 格式验证：MP4, MOV, AVI

#### 4. 分析进度界面（AnalysisProgressScreen）
**当前问题**：进度不可见，用户焦虑
**重构方案**：
- 大号圆形进度条（Neumorphic 风格）
- 分阶段说明：姿态识别 → 指标计算 → 评分生成
- 预计剩余时间
- 取消按钮（带二次确认）

#### 5. 结果展示界面（ResultScreen）
**当前问题**：信息层级不清晰
**重构方案**：
- 顶部：综合评分（大号显示，视觉突出）
- 中部：分项评分卡片（可滑动）
- 底部：关键帧轮播 + 改进建议
- 操作按钮：分享、保存、重新分析

#### 6. 历史记录界面（HistoryScreen）
**当前问题**：列表单调，对比困难
**重构方案**：
- 时间线布局，按日期分组
- 卡片展示：缩略图 + 评分 + 日期
- 筛选和排序功能
- 滑动操作：删除、对比

#### 7. 设置界面（SettingsScreen）
**重构方案**：
- 分组设置：通用、权限、存储、关于
- 清晰的开关和选项
- 存储空间管理：显示占用、清理缓存
- 关于页面：版本信息、帮助文档

### 性能优化策略

#### 1. 渲染优化
- 使用 `const` 构造函数减少重建
- `RepaintBoundary` 隔离复杂组件
- 懒加载历史记录列表

#### 2. 动画优化
- 使用 `AnimatedBuilder` 精确控制重绘范围
- 避免在 `build` 方法中创建动画
- 页面切换使用 `Hero` 动画

#### 3. 资源优化
- 图片资源压缩和缓存
- 视频缩略图异步加载
- 及时释放相机资源

### 测试策略

#### 1. 单元测试
- 设计系统组件测试
- 状态管理逻辑测试
- 工具函数测试

#### 2. Widget 测试
- 页面布局测试
- 交互行为测试
- 边界条件测试

#### 3. 集成测试

## Notes

### 设计参考
- 参考 Fitness App UI 模板的布局和交互模式
- 保持 Neumorphic 风格的一致性
- 注意深色模式的兼容性（未来扩展）

### 技术注意事项
- 相册权限需要在 Info.plist 和 AndroidManifest.xml 中配置
- Neumorphic 阴影效果可能影响性能，需要在真机测试
- 历史记录列表需要分页加载，避免一次性加载过多数据

### 用户体验注意事项
- 首次使用提供简短引导（不超过 3 步）
- 关键操作提供二次确认（删除、取消分析）
- 错误提示提供解决方案（如权限被拒绝时引导用户设置）
