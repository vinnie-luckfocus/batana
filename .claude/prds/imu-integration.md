# PRD: imu-integration

## Executive Summary

imu-integration 是 batana 项目的第三阶段交付，目标是通过集成棒尾 IMU（惯性测量单元）传感器，突破单目视觉的精度瓶颈，为专业球员与进阶用户提供高精度的挥棒分析能力。

为降低技术风险，V3 拆分为两个子阶段：
- **V3A（BLE 连接与基础指标）**：BLE 稳定连接、IMU 数据采集、基础指标输出——验证硬件通路可行性
- **V3B（融合算法与专业报告）**：视觉+IMU 融合、3D 轨迹重建、专业报告——建立精度壁垒

V3A 优先交付，确保 BLE 通路稳定后再投入融合算法开发。

## 阶段退出门槛（Go/No-Go）

V3 阶段的退出门槛（注：V4-MVP 可在 V3 完成前启动，不受 V3 阻塞）：

**V3A 退出门槛：**
1. IMU 配对成功率 ≥ 95%（P95）
2. BLE 连接稳定性：10 分钟会话断线率 < 5%
3. 基础 IMU 指标输出成功率 ≥ 90%
4. 黄金数据集采集完成（50+ 样本）

**V3B 退出门槛：**
1. 融合分析会话成功率 ≥ 90%
2. 融合分析耗时 P95 ≤ 8 秒
3. 专业报告导出率 ≥ 40%
4. 精度 KPI 达标（基于黄金数据集验证）

## Problem Statement

### 我们正在解决什么问题
V2 阶段的单目视觉分析存在固有局限：
1. 深度信息缺失，速度/轨迹为估算值，误差较大
2. 遮挡场景识别失败率高
3. 力量传递、加速度曲线等专业指标无法准确计算（V2 仅提供视觉代理指标）
4. 专业球员与教练对精度要求更高，现有方案不满足需求

### 为什么现在必须解决
1. V2 用户数据显示专业/进阶用户占比持续增长
2. 竞品开始引入传感器方案，需要建立差异化优势
3. IMU 硬件成本已降至可接受范围（< $50）
4. 融合分析是后续训练计划与教练模式的数据基础

## User Stories

### 目标用户
**Persona：专业球员/进阶训练者**
- 年龄：18-30 岁
- 场景：系统化训练，追求技术突破
- 需求：高精度数据、专业级分析、可量化的细微改进
- 痛点：视觉分析精度不足，无法支撑专业训练决策

### 核心用户故事

**US-1：连接 IMU 设备**
- 作为专业球员，我希望通过蓝牙快速连接棒尾 IMU，以便开始高精度分析。
- 验收标准：
  - 蓝牙扫描发现设备 ≤ 5 秒
  - 配对成功率 ≥ 95%
  - 连接状态实时显示（已连接/断开/信号弱/电量低）
  - 支持断线自动重连（3 次尝试，间隔 2/4/8 秒，总计最多 14 秒）

**US-2：实时数据预览**
- 作为专业球员，我希望在挥棒前看到 IMU 实时数据，以便确认设备工作正常。
- 验收标准：
  - 实时显示 3 轴角速度/加速度波形
  - 数据刷新率 ≥ 50 Hz
  - 异常数据告警（静止时漂移过大、数据中断）

**US-3：融合分析结果**（V3B）
- 作为专业球员，我希望看到融合视觉+IMU 的高精度分析结果。
- 验收标准：
  - 输出高精度挥棒速度、加速度峰值、角速度曲线、启动时序（核心指标）
  - 输出 3D 挥棒轨迹图、力量-时间曲线（扩展指标，视进度决定是否纳入）
  - 分析耗时 ≤ 8 秒
  - 所有指标标注置信度

**US-4：专业报告**（V3B）
- 作为专业球员，我希望获得详细的专业报告，以便与教练讨论或存档。
- 验收标准：
  - 包含所有专业指标与可视化图表
  - 支持导出 PDF/图片
  - 包含本次 vs 历史最佳对比

## Requirements

### Functional Requirements

#### V3A：BLE 连接与基础指标

#### FR-1 BLE 设备管理
- 蓝牙扫描与设备列表
- 设备配对与连接
- 连接状态监控（已连接/断开/信号弱/电量低）
- 断线自动重连（3 次尝试，指数退避：2/4/8 秒）
- 设备信息显示（电量、固件版本、信号强度）
- BLE 协议层要求：
  - 丢包率 < 1%（应用层校验）
  - 支持 MTU 协商（目标 ≥ 185 bytes）
  - 数据包序列号校验，检测丢包与乱序

#### FR-2 IMU 数据采集
- 实时接收 IMU 数据流：
  - 3 轴角速度（gyroscope）
  - 3 轴加速度（accelerometer）
  - 采样率：BLE 5.0 设备 ≥ 100 Hz；BLE 4.2 降级模式 ≥ 50 Hz（降低精度但保证可用性）
- 数据时间戳记录（毫秒级，基于设备本地时钟）
- 数据缓冲与本地存储（环形缓冲区，保留最近 60 秒）
- 数据压缩：16-bit 定点数编码，降低 BLE 传输负载

#### FR-3 设备校准
- 首次使用设备校准流程（引导式 UI）
- 静态校准（消除零漂，要求设备静止 5 秒）
- 动态校准（标定灵敏度，要求执行标准挥棒 3 次）
- 校准数据本地持久化，支持重新校准

#### FR-4 基础 IMU 指标（V3A 独立输出，不依赖融合）
- **挥棒速度**：IMU 角速度积分（棒头线速度估算）
- **加速度峰值**：3 轴加速度合成峰值及发生时刻
- **角速度峰值**：最大角速度及发生时刻
- 以上指标可独立于视觉分析输出，作为 V3A 的核心交付

#### V3B：融合算法与专业报告

#### FR-5 时序同步
- 视频帧时间戳与 IMU 数据时间戳对齐
- 时间偏移校准（通过挥棒启动瞬间的加速度突变对齐）
- 插值处理（视频 30 FPS vs IMU 100 Hz，线性插值）
- 时钟漂移补偿：每次会话开始时执行同步握手，记录偏移量
- 同步质量评估：输出同步置信度，低于阈值时提示用户重新录制

#### FR-6 数据融合算法
- 视觉轨迹 + IMU 轨迹融合（扩展卡尔曼滤波 EKF）
- 3D 轨迹重建（结合视觉 2D + IMU 深度信息）——基础管线必须实现，完整交付视扩展里程碑决定
- 速度/加速度曲线生成
- 力量传递效率计算——扩展指标，视进度决定是否纳入
- 融合算法参数可配置（滤波器增益、权重比例）

#### FR-7 专业指标计算
- **核心指标（V3B 必须交付）**：
  - **高精度挥棒速度**：融合视觉+IMU 的速度估算
  - **加速度峰值**：3 轴加速度合成峰值
  - **角速度曲线**：角速度-时间曲线
  - **启动时序**：髋/肩/腕启动时间差（毫秒级，视觉提供髋/肩，IMU 提供腕）
- **扩展指标（V3B 可选里程碑，视进度决定是否纳入）**：
  - **力量曲线**：力量-时间曲线
  - **3D 轨迹**：空间轨迹可视化
- 所有指标附带置信度评分

#### FR-8 专业报告生成
- 报告内容：
  - 综合评分 + 分项评分（融合版，置信度升级）
  - 所有核心指标数值与置信度
  - 加速度曲线图
  - 角速度曲线图
  - 对比分析（本次 vs 历史最佳）
  - 3D 轨迹图 + 力量曲线图（若扩展指标已实现）
- 支持导出 PDF/图片
- 报告模板可配置

#### FR-9 数据埋点与指标口径（V3）
- 核心事件埋点：
  - `imu_pair_start` / `imu_pair_success` / `imu_pair_failed`：IMU 配对
  - `imu_connect` / `imu_disconnect`：IMU 连接/断开
  - `imu_data_received`：IMU 数据接收成功（用于基础指标输出成功率）
  - `fusion_analysis_start` / `fusion_analysis_complete` / `fusion_analysis_failed`：融合分析（含时间戳）
  - `fusion_sync_error`：时序同步误差记录（毫秒级，每次融合分析记录）
  - `fusion_accuracy_test`：融合精度测试记录（仅在黄金数据集上运行，记录误差百分比）
  - `professional_report_export`：专业报告导出（V3 专用，区别于 V4 周期报告）
  - `app_open`：应用启动（继承自 MVP）
  - `user_first_open`：用户首次启动（继承自 MVP）
- KPI 计算口径：
  - **IMU 配对成功率（P95）** = P95(`imu_pair_success` / `imu_pair_start`)
  - **BLE 连接稳定性（10 分钟会话断线率）** = 10 分钟会话中触发 `imu_disconnect` 的会话数 / 总 10 分钟会话数
  - **基础 IMU 指标输出成功率** = `imu_data_received` 成功会话数 / `imu_connect` 成功会话数
  - **融合分析会话成功率** = `fusion_analysis_complete` / `fusion_analysis_start`
  - **融合分析耗时 P95** = P95(`fusion_analysis_complete` 时间戳 - `fusion_analysis_start` 时间戳)
  - **时序同步误差 P95** = P95(`fusion_sync_error` 记录值)
  - **3D 轨迹重建精度** = 基于黄金数据集的 `fusion_accuracy_test` 误差分布，V3B 启动前锁定目标值（如 P95 误差 < X%）
  - **专业报告导出率** = `professional_report_export` / `fusion_analysis_complete`
  - **IMU 用户 D30 留存率** = 第 30 天触发 `app_open` 的 IMU 用户数 / 第 1 天新增 IMU 用户数（IMU 用户定义：至少触发 1 次 `imu_pair_success`）
- 埋点实施责任：客户端 SDK

### Non-Functional Requirements

#### NFR-1 性能
- 融合分析耗时：P50 ≤ 5 秒，P95 ≤ 8 秒
- BLE 数据传输延迟 ≤ 50 ms（端到端）
- 配对成功率 ≥ 95%（P95，基于 100 次测试样本）
- 断线重连时间 ≤ 15 秒（含 3 次重试，间隔 2/4/8 秒）

#### NFR-2 稳定性
- 融合分析成功率 ≥ 90%（定义：IMU 已连接且视频质量合格的会话中成功输出结果的比例）
- BLE 连接稳定性：10 分钟会话断线率 < 5%
- 数据完整性 ≥ 99%（丢包率 < 1%，通过序列号校验）

#### NFR-3 兼容性
- 设备认证矩阵：
  - **A 级（完全支持）**：官方推荐设备，完整测试覆盖
  - **B 级（基本支持）**：通过基础协议测试的第三方设备
  - **C 级（实验性）**：未经测试，用户自行承担风险
- 兼容 iOS 14+ / Android 9+
- BLE 协议基线：目标 BLE 5.0，降级兼容 BLE 4.2（降低采样率至 50 Hz 以适配带宽）
- 固件版本策略：最低支持版本 + 推荐版本，不兼容时提示升级

#### NFR-4 可扩展性
- 模块化设计，BLE 协议层与业务逻辑解耦
- 融合算法参数可配置（不需重新编译）
- 传感器抽象层，便于新增传感器类型

## Success Criteria

### 业务指标
- IMU 配对成功率 ≥ 95%
- 融合分析会话成功率 ≥ 90%
- 专业报告导出率 ≥ 40%（定义：完成融合分析的用户中导出报告的比例）
- IMU 用户 D30 留存率 ≥ 30%

### 技术指标
- 融合分析耗时 P95 ≤ 8 秒
- 时序同步误差 P95 ≤ 20 ms（初始目标，基于黄金数据集验证后可收紧）
- 3D 轨迹重建精度：待黄金数据集建立后定义（见精度验证计划）

### 精度验证计划
精度 KPI 不预设绝对值，而是通过以下流程确定：
1. **黄金数据集建立**：使用高精度运动捕捉系统（如 Vicon/OptiTrack）采集 50+ 次标准挥棒数据
2. **基线测量**：在黄金数据集上运行融合算法，统计误差分布
3. **KPI 锁定**：基于 P50/P95 误差分布设定可达目标
4. **持续验证**：每次算法迭代在黄金数据集上回归测试

## Constraints & Assumptions

### 技术约束
1. BLE 传输带宽有限：目标协议基线 BLE 5.0（理论 2 Mbps），降级兼容 BLE 4.2（约 1 Mbps）；实际应用层吞吐约 10-20 KB/s（BLE 5.0）或 2-5 KB/s（BLE 4.2），需数据压缩
2. 时间同步依赖设备时钟精度，存在漂移风险
3. 融合算法需要真实数据训练与验证

### 资源约束
1. V3A 阶段 6-8 周交付，V3B 阶段 6-8 周交付
2. 需要硬件供应商协作（固件、协议文档）
3. 需要专业球员参与测试（至少 10 名，覆盖不同水平）

### 核心假设
1. 用户愿意为高精度分析购买 IMU 设备（< $50 价位）
2. BLE 连接稳定性在室外球场环境可接受
3. 融合算法精度提升足以支撑专业需求

## Out of Scope

V3 阶段明确不包含：
1. 多传感器同时连接（如多个 IMU）
2. 实时分析（仍为录制后分析）
3. 教练模式
4. 训练计划推荐
5. C 级设备的技术支持

## Dependencies

### 外部依赖
1. IMU 硬件供应商（固件、BLE 协议文档）
   - 备选供应商：至少评估 2 家，避免单点依赖
   - 固件版本管理：供应商需提供版本变更日志与向后兼容承诺

### 内部依赖
1. V2A 可视化能力已上线（软依赖，V3A 可独立验证 BLE 通路，但完整体验需骨骼叠加）
2. 黄金数据集采集完成（V3B 硬依赖，需在 V3A 结束前完成）
3. 专业球员测试资源就绪

### 黄金数据集采集计划
- **时机**：V3A 阶段并行进行，作为 V3A 退出门槛之一
- **规模**：50+ 次标准挥棒数据（使用 Vicon/OptiTrack 等高精度运动捕捉系统）
- **用途**：V3B 融合算法验证与精度 KPI 锁定
- **责任方**：产品团队协调专业球员 + 运动科学实验室
- **交付物**：标注数据集 + 基线误差报告

### 跨阶段依赖关系
- V3A → V2A：软依赖（V3A 可独立验证 BLE 通路，完整体验需 V2A 骨骼叠加）
- V3A → V2B：软依赖（云同步非必须）
- V3B → V3A：硬依赖（BLE 通路必须稳定 + 黄金数据集必须就绪）
- V3B → V2：硬依赖（需要 V2A + V2B 全量完成，确保可视化与云端数据基础设施就绪）
- V4 → V3：软依赖（V4-MVP 可基于 V2 数据先启动教练模式基础版）

## Risks & Mitigation

### 主要风险
1. **BLE 连接不稳定（室外环境干扰）** → 指数退避重连 + 本地数据缓冲；V3A 阶段重点压测室外场景
2. **时序同步误差大（时钟漂移）** → 每次会话同步握手 + 加速度突变对齐；输出同步置信度
3. **融合算法精度不足** → 不预设精度 KPI，通过黄金数据集实测后锁定；置信度标签管理预期
4. **硬件供应商单点依赖** → 评估至少 2 家供应商；协议层抽象，支持切换
5. **BLE 丢包导致数据不完整** → 序列号校验 + 丢包重传请求；丢包率 > 5% 时提示用户靠近设备
6. **固件兼容性问题** → 最低固件版本检测，不兼容时阻止连接并提示升级

### 缓解策略
1. V3A 阶段投入 50% 时间进行 BLE 稳定性压测（室内/室外/多设备干扰）
2. 建立标准测试数据集（黄金数据集）验证算法
3. 小规模试点（10 名专业球员）快速迭代

## 失败模式与降级策略

| 失败场景 | 降级策略 | 用户提示 |
|---------|---------|---------|
| BLE 配对失败 | 引导手动配对流程 | "自动配对失败，请在系统蓝牙设置中手动配对" |
| BLE 连接中断 | 自动重连 3 次；本地缓冲数据不丢失 | "连接中断，正在重连...（第 N 次）" |
| IMU 数据丢包 > 5% | 标记数据质量，降低置信度 | "数据传输不稳定，请靠近设备重试" |
| 时序同步失败 | 回退到仅视觉分析模式 | "时序同步失败，已使用视觉分析模式" |
| 融合分析失败 | 分别输出视觉结果和 IMU 基础指标 | "融合分析未成功，已分别展示视觉分析和传感器数据" |
| 设备固件不兼容 | 阻止连接 | "设备固件版本过低，请更新至 X.X.X 以上" |
| 校准失败 | 允许重试，最多 3 次 | "校准未通过，请确保设备静止后重试" |
