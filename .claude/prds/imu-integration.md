# PRD: imu-integration

## Executive Summary

imu-integration 是 batana 项目的第三阶段交付，目标是通过集成棒尾 IMU（惯性测量单元）传感器，突破单目视觉的精度瓶颈，为专业球员与进阶用户提供高精度的挥棒分析能力。核心技术是视觉关键点序列与 IMU 角速度/加速度数据的时序融合，输出 3D 轨迹、力量曲线、加速度峰值等专业级指标。

V3 阶段建立技术壁垒：BLE 稳定连接、毫秒级时间同步、卡尔曼滤波融合算法、专业报告生成。分析精度从"可用"提升至"专业级"，分析耗时优化至 8 秒以内，为后续教练模式与训练计划奠定数据基础。

## Problem Statement

### 我们正在解决什么问题
V2 阶段的单目视觉分析存在固有局限：
1. 深度信息缺失，速度/轨迹为估算值，误差较大
2. 遮挡场景识别失败率高
3. 力量传递、加速度曲线等专业指标无法准确计算
4. 专业球员与教练对精度要求更高，现有方案不满足需求

### 为什么现在必须解决
1. V2 用户数据显示专业/进阶用户占比持续增长
2. 竞品开始引入传感器方案，需要建立差异化优势
3. IMU 硬件成本已降至可接受范围（< $50）
4. 融合分析是后续训练计划与教练模式的数据基础

## User Stories

### 目标用户
**Persona：专业球员/进阶训练者**
- 年龄：18-30 岁
- 场景：系统化训练，追求技术突破
- 需求：高精度数据、专业级分析、可量化的细微改进
- 痛点：视觉分析精度不足，无法支撑专业训练决策

### 核心用户故事

**US-1：连接 IMU 设备**
- 作为专业球员，我希望通过蓝牙快速连接棒尾 IMU，以便开始高精度分析。
- 验收标准：
  - 蓝牙扫描发现设备 ≤ 5 秒
  - 配对成功率 ≥ 95%
  - 连接状态实时显示
  - 支持断线自动重连

**US-2：实时数据预览**
- 作为专业球员，我希望在挥棒前看到 IMU 实时数据（角速度/加速度），以便确认设备工作正常。
- 验收标准：
  - 实时显示 3 轴角速度/加速度
  - 数据刷新率 ≥ 50 Hz
  - 异常数据告警

**US-3：融合分析结果**
- 作为专业球员，我希望看到融合视觉+IMU 的高精度分析结果，包括 3D 轨迹、力量曲线、加速度峰值。
- 验收标准：
  - 输出 3D 挥棒轨迹图
  - 输出力量-时间曲线
  - 输出加速度峰值与发生时刻
  - 分析耗时 ≤ 8 秒

**US-4：专业报告**
- 作为专业球员，我希望获得详细的专业报告（PDF/图片），以便与教练讨论或存档。
- 验收标准：
  - 包含所有专业指标
  - 包含可视化图表
  - 支持导出分享

## Requirements

### Functional Requirements

#### FR-1 BLE 设备管理
- 蓝牙扫描与设备列表
- 设备配对与连接
- 连接状态监控（已连接/断开/信号弱）
- 断线自动重连（3 次尝试）
- 设备信息显示（电量、固件版本）

#### FR-2 IMU 数据采集
- 实时接收 IMU 数据流：
  - 3 轴角速度（gyroscope）
  - 3 轴加速度（accelerometer）
  - 采样率 ≥ 100 Hz
- 数据时间戳记录（毫秒级）
- 数据缓冲与存储

#### FR-3 时序同步
- 视频帧时间戳与 IMU 数据时间戳对齐
- 时间偏移校准（通过挥棒启动瞬间对齐）
- 插值处理（视频 30 FPS vs IMU 100 Hz）

#### FR-4 数据融合算法
- 视觉轨迹 + IMU 轨迹融合（卡尔曼滤波）
- 3D 轨迹重建（结合视觉 2D + IMU 深度信息）
- 速度/加速度曲线生成
- 力量传递效率计算

#### FR-5 专业指标计算
- **高精度挥棒速度**：IMU 角速度积分
- **加速度峰值**：3 轴加速度合成峰值
- **力量曲线**：力量-时间曲线
- **3D 轨迹**：空间轨迹可视化
- **角速度曲线**：角速度-时间曲线
- **启动时序**：髋/肩/腕启动时间差（毫秒级）

#### FR-6 专业报告生成
- 报告内容：
  - 综合评分 + 分项评分
  - 所有专业指标数值
  - 3D 轨迹图
  - 力量曲线图
  - 加速度曲线图
  - 对比分析（本次 vs 历史最佳）
- 支持导出 PDF/图片

#### FR-7 设备校准
- 首次使用设备校准流程
- 静态校准（消除零漂）
- 动态校准（标定灵敏度）

### Non-Functional Requirements

#### NFR-1 性能
- 融合分析耗时 ≤ 8 秒
- BLE 数据传输延迟 ≤ 50 ms
- 配对成功率 ≥ 95%
- 断线重连时间 ≤ 3 秒

#### NFR-2 稳定性
- 融合分析成功率 ≥ 90%
- BLE 连接稳定性（10 分钟无断线）
- 数据同步准确率 ≥ 95%

#### NFR-3 兼容性
- 支持多款 IMU 设备（可扩展协议）
- 兼容 iOS/Android BLE 标准

#### NFR-4 可扩展性
- 模块化设计，便于新增传感器类型
- 融合算法可配置参数

## Success Criteria

### 业务指标
- IMU 配对成功率 ≥ 95%
- 融合分析会话成功率 ≥ 90%
- 专业/进阶用户月活占比持续增长
- 专业报告导出率 ≥ 40%

### 技术指标
- 融合分析耗时 ≤ 8 秒
- 时序同步误差 ≤ 10 ms
- 3D 轨迹重建精度（与标准设备对比误差 < 5%）

## Constraints & Assumptions

### 技术约束
1. BLE 传输带宽有限，需要数据压缩
2. 时间同步依赖设备时钟精度
3. 融合算法需要大量真实数据训练

### 资源约束
1. V3 阶段 3-4 个月交付
2. 需要硬件供应商协作（固件、协议）
3. 需要专业球员参与测试

### 核心假设
1. 用户愿意为高精度分析购买 IMU 设备
2. BLE 连接稳定性在实际使用场景可接受
3. 融合算法精度提升足以支撑专业需求

## Out of Scope

V3 阶段明确不包含：
1. 多传感器同时连接（如多个 IMU）
2. 实时分析（仍为录制后分析）
3. 教练模式
4. 训练计划推荐
5. 与第三方设备的全面兼容（优先支持官方推荐设备）

## Dependencies

### 外部依赖
1. IMU 硬件供应商（固件、BLE 协议文档）
2. BLE Flutter 插件（flutter_blue_plus）
3. 数学库（矩阵运算、滤波算法）

### 内部依赖
1. V2 可视化能力已上线
2. 融合算法设计与验证完成
3. 专业球员测试资源

## Risks & Mitigation

### 主要风险
1. BLE 连接不稳定 → 建立重连机制与离线缓存
2. 时序同步误差大 → 多次校准与手动对齐辅助
3. 融合算法精度不足 → 建立可解释性与置信度指标
4. 硬件供应链风险 → 支持多款设备协议

### 缓解策略
1. 提前进行 BLE 稳定性压测
2. 建立标准测试数据集验证算法
3. 小规模试点快速迭代
