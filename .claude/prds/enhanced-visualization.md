# PRD: enhanced-visualization

## Executive Summary

enhanced-visualization 是 batana 项目的第二阶段交付，目标是在 MVP 验证核心价值后，通过可视化增强与数据分析能力提升用户体验与留存率。核心策略是让分析结果"看得见、看得懂、可追踪"。

为降低交付风险，V2 拆分为两个子阶段：
- **V2A（可视化与解释）**：骨骼叠加、关键帧慢放、分项评分、本地历史趋势——聚焦核心留存功能
- **V2B（账号与云同步）**：账号体系、云端存储、跨设备同步——聚焦基础设施建设

V2A 优先上线，确保核心体验优化不被后端基础设施开发拖慢。

## 阶段退出门槛（Go/No-Go）

### V2A 启动门槛（允许 V3A 并行启动）
V2A 核心功能上线后即可启动 V3A（BLE 通路验证），无需等待 V2B：
1. 骨骼叠加渲染流畅度 ≥ 30 FPS
2. 分项评分功能上线
3. 崩溃率 < 0.5%

### V2 全量收官门槛（进入 V3B 的前提）
V2A + V2B 全部上线后方可进入 V3B：
1. 分析完成率 ≥ 90%
2. D30 留存率 ≥ 20%
3. 分项评分查看率 ≥ 70%
4. 崩溃率 < 0.5%
5. 云同步成功率 ≥ 99%（P95）

## Problem Statement

### 我们正在解决什么问题
MVP 验证了基础需求，但用户反馈暴露新痛点：
1. 综合评分不够直观，用户不知道"为什么是这个分数"
2. 文字建议难以理解，需要可视化辅助
3. 无法精准定位"问题发生在哪一瞬间"
4. 缺乏长期进步追踪，难以保持训练动力
5. 换设备后历史数据丢失

### 为什么现在必须解决
1. MVP 用户留存数据显示"可视化"是高频需求
2. 竞品已有骨骼叠加功能，需要快速跟进
3. 云同步是用户长期使用的基础设施

## User Stories

### 目标用户
**Persona：青少年球员（进阶训练）**
- 年龄：12-18 岁
- 场景：每周固定训练，关注技术细节与长期进步
- 需求：看到动作细节、追踪训练效果、跨设备查看数据
- 痛点：动作时好时坏，不知道哪个环节需要重点改进

### 核心用户故事

**US-1：可视化动作问题**
- 作为进阶球员，我希望看到骨骼叠加在视频上，以便直观看到身体各部位的位置关系。
- 验收标准：
  - 视频回放时叠加骨骼线条
  - 关键关节点清晰标注
  - 支持暂停查看任意帧

**US-2：关键帧慢放**
- 作为进阶球员，我希望看到准备/挥击/收尾三个关键阶段的慢动作，以便精准定位问题发生时刻。
- 验收标准：
  - 自动识别 3 个关键帧
  - 支持 0.25x 慢放回放
  - 可手动切换关键帧

**US-3：分项评分**
- 作为进阶球员，我希望看到姿态、速度、动作效率、时机、流畅度的分项评分，以便明确优先改进方向。
- 验收标准：
  - 提供 5 个分项评分
  - 每项配简短说明与置信度标注
  - 标注最弱项

**US-4：历史趋势**
- 作为进阶球员，我希望看到最近 30 次训练的评分趋势图，以便量化进步。
- 验收标准：
  - 折线图展示综合评分趋势
  - 可切换查看分项评分趋势
  - 显示最高分/最低分/平均分

**US-5：跨设备同步**（V2B）
- 作为进阶球员，我希望在手机和平板上都能查看训练历史，以便随时复盘。
- 验收标准：
  - 支持账号登录（邮箱 + 第三方登录）
  - 训练记录自动云端同步
  - 换设备后数据完整保留

## Requirements

### Functional Requirements

#### V2A：可视化与分项评分

#### FR-1 骨骼叠加可视化
- 视频回放时叠加 2D 骨骼线条
- 关键关节点用圆点标注
- 支持骨骼显示/隐藏切换
- 骨骼颜色可配置（默认绿色）
- 使用 GPU 硬件加速渲染，确保流畅度

#### FR-2 关键帧检测与慢放
- 自动识别 3 个关键帧：
  - 准备期结束（启动瞬间）
  - 击球期（速度峰值）
  - 收尾期结束
- 支持 0.25x / 0.5x / 1x 速度切换
- 支持手动拖动进度条定位

#### FR-3 分项评分体系
- 5 个分项评分维度（均基于视觉分析，标注为"视觉代理指标"）：
  - **姿态评分**：站位、重心、身体角度（置信度：高）
  - **速度评分**：挥棒速度峰值的相对评级（置信度：中）
  - **动作效率评分**：加速度曲线平滑度、动作幅度合理性（置信度：中，标注"精确力量数据需 IMU 传感器"）
  - **时机评分**：髋肩分离时序、启动时机（置信度：高）
  - **流畅度评分**：动作连贯性、节奏稳定性（置信度：高）
- 每项 0-100 分，配简短说明 + 置信度标签（高/中/低）
- 综合评分 = 加权平均（权重通过配置文件管理）
- 置信度为"中"或"低"的指标在 UI 上以浅色/虚线样式区分

#### FR-4 历史趋势分析（V2A 阶段本地存储，V2B 阶段迁移至云端）
- 存储最近 30 次训练记录
- 趋势图：
  - 折线图展示综合评分
  - 可切换查看 5 个分项评分
  - 显示统计数据（最高/最低/平均/标准差）
- 支持按日期筛选

#### FR-5 结果页增强
- 新增分项评分卡片（含置信度标签）
- 新增"查看回放"按钮（跳转骨骼叠加视频）
- 新增"查看趋势"按钮（跳转历史趋势页）

#### FR-9 数据埋点与指标口径（V2）
- 核心事件埋点：
  - `app_open`：应用启动（继承自 MVP）
  - `user_first_open`：用户首次启动（继承自 MVP）
  - `analysis_start` / `analysis_complete`：分析开始/成功（继承自 MVP，含时间戳）
  - `video_record_complete`：录制完成（继承自 MVP）
  - `result_view`：查看分析结果（继承自 MVP）
  - `app_crash`：应用崩溃（继承自 MVP）
  - `skeleton_view`：查看骨骼叠加视频
  - `skeleton_render_fps`：骨骼渲染帧率采样（每秒采样 1 次，记录当前 FPS）
  - `detailed_score_view`：查看分项评分
  - `trend_view`：查看历史趋势
  - `account_register`：账号注册（V2B）
  - `sync_success` / `sync_failed`：云同步成功/失败（V2B）
- KPI 计算口径：
  - **分析完成率** = `analysis_complete` / `video_record_complete`
  - **分析耗时 P95** = P95(`analysis_complete` 时间戳 - `analysis_start` 时间戳)
  - **骨骼叠加渲染流畅度** = `skeleton_render_fps` 采样值的 P50（中位数）
  - **D30 留存率** = 第 30 天触发 `app_open` 的用户数 / 第 1 天触发 `user_first_open` 的用户数
  - **分项评分查看率** = `detailed_score_view` / `result_view`
  - **历史趋势功能使用率** = 有 ≥3 次记录且触发 `trend_view` 的用户数 / 有 ≥3 次记录的用户数
  - **云同步成功率（P95）** = P95(按用户维度统计：单用户 `sync_success` / (`sync_success` + `sync_failed`))
  - **云同步激活率** = 注册账号且至少触发 1 次 `sync_success` 的用户数 / 总用户数
  - **崩溃率** = `app_crash` 会话数 / 总会话数
- 埋点实施责任：客户端 SDK + 服务端日志

#### V2B：账号与云同步

#### FR-6 账号体系
- 账号注册/登录（邮箱 + 密码）
- 第三方登录：Google / Apple Sign-In
- 账号注销与数据删除（GDPR/隐私合规）
- 未成年人保护：13 岁以下用户需家长同意（通过家长邮箱验证）

#### FR-7 云端同步
- 训练记录自动云端同步（仅同步元数据与评分，不同步原始视频）
- 支持手动触发同步
- 离线模式：本地缓存，联网后自动同步
- 数据一致性模型：
  - **权威源**：云端为权威源（Server-Authoritative）
  - **冲突解决**：记录级 Last-Write-Wins（LWW），以服务端接收时间戳为准
  - **本地角色**：本地 SQLite 为缓存层，离线时可读写，联网后推送至云端
  - **冲突场景**：同一记录在多设备同时修改时，服务端保留最后到达的版本，覆盖先到达的版本
  - **冲突日志**：服务端记录冲突事件，便于排查（保留 30 天）
- 同步失败时本地数据不丢失，下次联网自动重试（最多 3 次，之后提示用户手动同步）

### Non-Functional Requirements

#### NFR-1 性能
- 分析结果生成时间：P50 ≤ 8 秒，P95 ≤ 10 秒（基准机型同 MVP）
- 骨骼叠加渲染流畅度 ≥ 30 FPS（GPU 加速）
- 趋势图加载时间 ≤ 2 秒

#### NFR-2 可用性
- 分项评分查看率 ≥ 70%（定义：查看分析结果的用户中点击分项评分的比例）
- 历史趋势功能使用率 ≥ 50%（定义：有 3 次以上记录的用户中使用趋势图的比例）
- 云同步成功率：P95 ≥ 99%，P99 ≥ 97%

#### NFR-3 数据一致性
- 跨设备数据同步延迟：P95 ≤ 5 秒
- 数据一致性保证：
  - 云端为权威源，客户端最终一致性（Eventual Consistency）
  - 同步冲突按记录级 LWW 解决
  - 冲突率 < 1%（日活用户）
- 离线数据零丢失（本地缓存写入成功即视为持久化成功）

#### NFR-4 隐私与安全
- 用户数据加密传输（TLS 1.3）
- 云端数据 AES-256 加密存储
- 支持账号注销与完整数据删除（30 天内完成）
- 未成年人数据保护：不采集非必要个人信息，家长可代为删除

#### NFR-5 合规
- 隐私政策与用户协议（中英文）
- 13 岁以下用户家长同意机制（COPPA 合规）：
  - 可验证家长同意（VPC）方案：
    - **V2 阶段**：Email Plus 方式（家长邮箱验证 + 在线确认），适用于应用自身数据采集与处理，不涉及第三方披露
    - **注**：若后续涉及第三方数据披露（如 V4 教练访问），需升级为更强 VPC 方式（见 V4 PRD）
  - 同意证据留存 3 年
  - 家长可随时撤回同意并删除数据
- 数据导出功能（用户可导出自己的全部数据，JSON 格式）

## Constraints & Assumptions

### 技术约束
1. 骨骼叠加需要视频帧与关键点时序对齐
2. 云端仅存储元数据与评分（不存储原始视频），控制存储成本
3. 分项评分中"动作效率"为视觉代理指标，精确力量数据需等待 V3 IMU 集成

### 资源约束
1. V2A 阶段 6-8 周交付，V2B 阶段 4-6 周交付
2. V2B 需要后端开发资源（账号、存储 API）

### 核心假设
1. 用户愿意注册账号以获得云同步能力
2. 分项评分的置信度标注足以管理用户预期
3. 30 次历史记录足够满足趋势分析需求

## Out of Scope

V2 阶段明确不包含：
1. IMU 传感器集成
2. 3D 轨迹可视化
3. 专业球员示范对比
4. 社交分享与排行榜
5. 教练模式
6. 付费订阅功能
7. 实时视频通话

## Dependencies

### 外部依赖
1. 视频编解码库（用于骨骼叠加渲染，如 ffmpeg_kit_flutter）
2. 图表库（用于趋势图，如 fl_chart）
3. 云服务基础设施（V2B）：
   - **主方案**：Firebase（Auth + Firestore + Cloud Functions）——生态成熟、Flutter 集成度高
   - **备选方案**：Supabase（Auth + PostgreSQL + Edge Functions）——开源、可自托管
   - **选型决策时机**：V2B 立项前（V2A 开发期间完成 PoC 对比评估）
   - **评估标准**：Flutter SDK 成熟度、离线同步支持、成本模型、数据导出便利性

### 内部依赖
1. MVP 核心分析能力已上线且通过 Go/No-Go 门槛（硬依赖）
2. 分项评分算法设计完成（V2A 硬依赖）
3. 后端 API 开发（V2B 硬依赖，不阻塞 V2A）

### 跨阶段依赖关系
- V2A → MVP：硬依赖（MVP 必须通过退出门槛）
- V2B → V2A：软依赖（V2B 可与 V2A 并行开发，但上线需在 V2A 之后）
- V3A → V2A：软依赖（V3A 可独立验证 BLE 通路，完整体验需 V2A 骨骼叠加）
- V3B → V2：硬依赖（V3B 需要 V2A + V2B 全量完成，确保可视化与云端数据基础设施就绪）

## Risks & Mitigation

### 主要风险
1. **骨骼叠加性能问题** → 使用 GPU 硬件加速渲染；降级方案：仅显示关键帧静态骨骼图
2. **云同步成本过高** → 仅同步元数据与评分，不同步原始视频；设置免费额度上限
3. **分项评分可信度质疑** → 置信度标签管理预期；"动作效率"明确标注为代理指标
4. **V2B 后端开发阻塞** → V2A 不依赖后端，可独立上线；V2B 延期不影响核心体验
5. **离线/在线数据冲突** → LWW 策略 + 冲突日志记录，便于排查
6. **未成年人合规风险** → 家长邮箱验证机制；不采集非必要 PII

### 缓解策略
1. 提前进行骨骼渲染性能压测（目标机型矩阵）
2. 云存储成本模型预估（按 MAU 1000/5000/10000 分级）
3. 灰度发布 V2A 收集反馈后再启动 V2B

## 失败模式与降级策略

| 失败场景 | 降级策略 | 用户提示 |
|---------|---------|---------|
| 骨骼渲染卡顿 | 降低渲染帧率至 15 FPS 或仅显示关键帧静态图 | "设备性能不足，已切换为关键帧模式" |
| 关键帧检测失败 | 均匀取 3 帧作为替代 | "自动检测未成功，已选取代表帧" |
| 云同步失败 | 本地数据保留，标记待同步 | "同步失败，数据已保存在本地，稍后自动重试" |
| 第三方登录不可用 | 回退到邮箱登录 | "第三方登录暂不可用，请使用邮箱登录" |
| 趋势图数据不足 | 显示已有数据 + 提示 | "需要至少 3 次训练记录才能生成趋势图" |
