# PRD: enhanced-visualization

## Executive Summary

enhanced-visualization 是 batana 项目的第二阶段交付，目标是在 MVP 验证核心价值后，通过可视化增强与数据分析能力提升用户体验与留存率。核心策略是让分析结果"看得见、看得懂、可追踪"：骨骼叠加让用户直观看到动作问题，关键帧慢放精准定位错误瞬间，分项评分明确改进方向，历史趋势图量化进步，云端同步支持跨设备使用。

V2 阶段在 MVP 基础上扩展 5 个分项评分维度、30 次历史趋势分析、账号体系与云端存储，同时优化分析性能至 10 秒以内，为专业用户与长期训练场景提供更深度的价值。

## Problem Statement

### 我们正在解决什么问题
MVP 验证了基础需求，但用户反馈暴露新痛点：
1. 综合评分不够直观，用户不知道"为什么是这个分数"
2. 文字建议难以理解，需要可视化辅助
3. 无法精准定位"问题发生在哪一瞬间"
4. 缺乏长期进步追踪，难以保持训练动力
5. 换设备后历史数据丢失

### 为什么现在必须解决
1. MVP 用户留存数据显示"可视化"是高频需求
2. 竞品已有骨骼叠加功能，需要快速跟进
3. 云同步是用户长期使用的基础设施

## User Stories

### 目标用户
**Persona：青少年球员（进阶训练）**
- 年龄：12-18 岁
- 场景：每周固定训练，关注技术细节与长期进步
- 需求：看到动作细节、追踪训练效果、跨设备查看数据
- 痛点：动作时好时坏，不知道哪个环节需要重点改进

### 核心用户故事

**US-1：可视化动作问题**
- 作为进阶球员，我希望看到骨骼叠加在视频上，以便直观看到身体各部位的位置关系。
- 验收标准：
  - 视频回放时叠加骨骼线条
  - 关键关节点清晰标注
  - 支持暂停查看任意帧

**US-2：关键帧慢放**
- 作为进阶球员，我希望看到准备/挥击/收尾三个关键阶段的慢动作，以便精准定位问题发生时刻。
- 验收标准：
  - 自动识别 3 个关键帧
  - 支持 0.25x 慢放回放
  - 可手动切换关键帧

**US-3：分项评分**
- 作为进阶球员，我希望看到姿态、速度、力量、时机、流畅度的分项评分，以便明确优先改进方向。
- 验收标准：
  - 提供至少 5 个分项评分
  - 每项配简短说明
  - 标注最弱项

**US-4：历史趋势**
- 作为进阶球员，我希望看到最近 30 次训练的评分趋势图，以便量化进步。
- 验收标准：
  - 折线图展示综合评分趋势
  - 可切换查看分项评分趋势
  - 显示最高分/最低分/平均分

**US-5：跨设备同步**
- 作为进阶球员，我希望在手机和平板上都能查看训练历史，以便随时复盘。
- 验收标准：
  - 支持账号登录
  - 训练记录自动云端同步
  - 换设备后数据完整保留

## Requirements

### Functional Requirements

#### FR-1 骨骼叠加可视化
- 视频回放时叠加 2D 骨骼线条
- 关键关节点用圆点标注
- 支持骨骼显示/隐藏切换
- 骨骼颜色可配置（默认绿色）

#### FR-2 关键帧检测与慢放
- 自动识别 3 个关键帧：
  - 准备期结束（启动瞬间）
  - 击球期（速度峰值）
  - 收尾期结束
- 支持 0.25x / 0.5x / 1x 速度切换
- 支持手动拖动进度条定位

#### FR-3 分项评分体系
- 5 个分项评分维度：
  - **姿态评分**：站位、重心、身体角度
  - **速度评分**：挥棒速度峰值
  - **力量评分**：加速度曲线、力量传递效率
  - **时机评分**：髋肩分离时序、启动时机
  - **流畅度评分**：动作连贯性、节奏稳定性
- 每项 0-100 分，配简短说明
- 综合评分 = 加权平均

#### FR-4 历史趋势分析
- 存储最近 30 次训练记录（云端）
- 趋势图：
  - 折线图展示综合评分
  - 可切换查看 5 个分项评分
  - 显示统计数据（最高/最低/平均/标准差）
- 支持按日期筛选

#### FR-5 账号与云端同步
- 账号注册/登录（邮箱 + 密码）
- 训练记录自动云端同步
- 支持手动触发同步
- 离线模式：本地缓存，联网后自动同步

#### FR-6 结果页增强
- 新增分项评分卡片
- 新增"查看回放"按钮（跳转骨骼叠加视频）
- 新增"查看趋势"按钮（跳转历史趋势页）

### Non-Functional Requirements

#### NFR-1 性能
- 分析结果生成时间 ≤ 10 秒（优化 5 秒）
- 骨骼叠加渲染流畅度 ≥ 30 FPS
- 趋势图加载时间 ≤ 2 秒

#### NFR-2 可用性
- 分项评分查看率 ≥ 70%
- 历史趋势功能使用率 ≥ 50%
- 云同步成功率 ≥ 99%

#### NFR-3 数据一致性
- 跨设备数据同步延迟 ≤ 5 秒
- 离线数据不丢失

#### NFR-4 隐私与安全
- 用户数据加密传输（HTTPS）
- 云端数据加密存储
- 支持账号注销与数据删除

## Success Criteria

### 业务指标
- 分析完成率 ≥ 90%
- D30 留存率 ≥ 20%
- 历史复盘功能使用率 ≥ 50%
- 分项评分查看率 ≥ 70%
- 云同步激活率 ≥ 60%

### 技术指标
- 分析耗时 ≤ 10 秒
- 云同步成功率 ≥ 99%
- 崩溃率 < 0.5%

## Constraints & Assumptions

### 技术约束
1. 骨骼叠加需要视频帧与关键点时序对齐
2. 云端存储成本需控制（视频压缩或仅存关键帧）
3. 分项评分算法需要更多训练数据验证

### 资源约束
1. V2 阶段 2-3 个月交付
2. 需要后端开发资源（账号、存储 API）

### 核心假设
1. 用户愿意注册账号以获得云同步能力
2. 分项评分可解释性足以支撑用户理解
3. 30 次历史记录足够满足趋势分析需求

## Out of Scope

V2 阶段明确不包含：
1. IMU 传感器集成
2. 3D 轨迹可视化
3. 专业球员示范对比
4. 社交分享与排行榜
5. 教练模式
6. 付费订阅功能

## Dependencies

### 外部依赖
1. 云服务基础设施（Firebase 或自建）
2. 视频编解码库（用于骨骼叠加渲染）
3. 图表库（用于趋势图）

### 内部依赖
1. MVP 核心分析能力已上线
2. 分项评分算法设计完成
3. 后端 API 开发（账号、存储）

## Product Roadmap

### M2：分项评分体系（2 周）

- 趋势图 UI

- 云端 API 开发

### M5：测试与优化（2 周）
- 性能优化
- 云同步稳定性测试
- 用户测试

## Risks & Mitigation

### 主要风险
1. 骨骼叠加性能问题 → 使用硬件加速渲染
2. 云同步成本过高 → 仅同步元数据，视频按需下载
3. 分项评分不可信 → 建立评分可解释性文档

### 缓解策略
1. 提前进行性能压测
2. 设计合理的数据存储策略
3. 灰度发布收集反馈
